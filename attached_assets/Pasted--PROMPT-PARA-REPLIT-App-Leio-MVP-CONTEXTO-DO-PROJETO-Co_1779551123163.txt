# PROMPT PARA REPLIT — App "Leio" (MVP)

---

## CONTEXTO DO PROJETO

Construa um aplicativo web mobile-first (PWA) chamado **"Leio"** — um app de leitura gamificado inspirado no Strava. O conceito central: transformar sessões de leitura em conquistas visuais compartilháveis nas redes sociais. O público é brasileiro, tudo deve estar em português do Brasil.

A proposta de valor é: "Se não registrou, não leu." O app deve tornar a leitura um ato social e visualmente orgulhoso, não uma atividade solitária e invisível.

O diferencial competitivo em relação ao Bookly (concorrente direto) é: a imagem compartilhável é o produto, não uma feature secundária. O card de conclusão de sessão precisa ser tão bonito e reconhecível que funcione como marketing orgânico toda vez que alguém postar no Instagram/Stories.

---

## STACK TÉCNICA

- **Frontend:** React + TypeScript + Tailwind CSS
- **Estrutura:** PWA (Progressive Web App) com manifest.json e service worker
- **Backend:** Node.js + Express (ou Fastify)
- **Banco de dados:** SQLite (via better-sqlite3 ou Prisma com SQLite) para o MVP
- **API de livros:** Google Books API (gratuita, busca por ISBN e título, retorna capa, autor, gênero, número de páginas)
- **Geração de imagem:** html2canvas ou dom-to-image para gerar o card compartilhável como PNG
- **Scan de código de barras:** biblioteca QuaggaJS ou html5-qrcode para leitura de ISBN via câmera
- **Armazenamento de imagens:** Local/base64 no MVP
- **Autenticação:** Simples com email + senha (bcrypt + JWT) — permitir uso sem login até o momento de compartilhar

## ARQUITETURA DE TELAS

**Cabeçalho:**
- Saudação personalizada: "E aí, [nome]! Bora ler?" ou saudação por horário (Bom dia/Boa tarde/Boa noite)
- Avatar/iniciais do usuário
- Ícone de streak (chama de fogo) com número de dias consecutivos lendo

**Seção "Leitura atual":**
- Card grande com a capa do livro atual, título, autor
- Barra de progresso (páginas lidas / total de páginas)

**Seção "Suas conquistas recentes":**
- Últimos 3 badges desbloqueados em formato horizontal scrollável

**Seção "Estatísticas rápidas":**
- Grid 2x2 com metric cards:

### 3. TELA DE NOVA SESSÃO — SELECIONAR LIVRO
Se o usuário já tem um livro em leitura, pré-selecionar automaticamente.

**Opções para adicionar livro:**
- Campo de busca por título/autor (consulta Google Books API)
- Botão "Escanear código de barras" (abre câmera, lê ISBN, busca na API)
- Lista dos livros já na biblioteca do usuário

**Card de resultado da busca:**
- Capa do livro (thumbnail da API)
- Título, autor, número de páginas, gênero/categoria
- Botão "Selecionar este livro"

**Ao selecionar:**
- Perguntar "Em que página você está?" (campo numérico)
- Livro adicionado à biblioteca automaticamente
- Ir para a tela de sessão ativa

### 4. TELA DE SESSÃO ATIVA (o coração do app)
Design inspirado no timer do Strava durante atividade.

**Layout:**
- Capa do livro no topo (pequena, com título e autor)
- Timer grande no centro (MM:SS), fonte monospace, animação de pulsação sutil a cada segundo
- Pace em tempo real abaixo do timer: "0.0 pág/min" (atualiza conforme o usuário registra páginas)
- Botão "Atualizar página atual" — abre um input numérico rápido para o usuário informar em que página está (cada atualização recalcula o pace em tempo real)
- Página inicial da sessão (fixa, definida no início)
- Página atual (editável durante a sessão)
- Páginas lidas na sessão (calculado automaticamente: página atual - página inicial)

**Controles:**
- Botão "Pausar" (pausa o timer, permite retomar)
- Botão "Encerrar sessão" (finaliza e vai para a tela de geração do card)
- O timer deve funcionar mesmo com a tela bloqueada/em background (usar Web Workers ou service worker para manter contagem)

**Informação secundária (colapsável):**
- Progresso geral do livro (barra de progresso)
- Estimativa de tempo para concluir o livro baseada no pace atual

### 5. TELA DE CONCLUSÃO DA SESSÃO — GERAÇÃO DO CARD COMPARTILHÁVEL
Esta é a tela mais importante do app. É o momento "troféu".

**Resumo da sessão (topo):**

**Se o usuário desbloqueou um badge nesta sessão:**
- Animação especial de badge desbloqueado (confetti + badge girando)

**Geração do card compartilhável:**

O card é uma imagem de 1080x1920px (formato Stories) ou 1080x1080px (formato feed).

**Elementos do card:**
- Imagem de fundo: o usuário pode escolher entre:

## LÓGICA DE NEGÓCIO IMPORTANTE

### Cálculo do Pace
- Pace = páginas lidas ÷ (duração em minutos)
- Exibir com 1 casa decimal: "1.3 pág/min"
- Pace em tempo real durante a sessão: atualizar a cada vez que o usuário informar a página atual
- Pace médio do livro: média ponderada por duração de todas as sessões do livro
- Pace médio geral: média ponderada de todas as sessões do usuário

### Cálculo do Streak
- Após cada sessão, verificar se o usuário leu no dia anterior
- Se sim, incrementar streak_current
- Se não (pulou um dia), resetar streak_current para 1
- Atualizar streak_max se streak_current > streak_max
- Considerar o dia pela data local do usuário (timezone Brasil: UTC-3)

### Verificação de Badges
Após cada sessão salva, rodar a verificação completa de badges:
1. Verificar todos os critérios de badges que o usuário ainda não desbloqueou
2. Para cada badge desbloqueado, salvar em user_badges com o session_id que triggered
3. Retornar lista de badges recém-desbloqueados para a tela de conclusão

### Busca de livros (Google Books API)
- Endpoint: `GET https://www.googleapis.com/books/v1/volumes?q={query}`
- Para ISBN: `GET https://www.googleapis.com/books/v1/volumes?q=isbn:{isbn}`
- Extrair: title, authors[0], pageCount, categories[0], imageLinks.thumbnail, industryIdentifiers
- Se thumbnail vier como http, converter para https
- Tratar casos onde pageCount é undefined (pedir ao usuário para informar manualmente)

### Geração do card compartilhável
- Renderizar um componente React invisível (off-screen) com o layout do card
- Usar html2canvas para converter em PNG
- Tamanho: 1080x1920 (Stories) e 1080x1080 (Feed)
- O componente do card deve usar posicionamento absoluto e fontes inline (Google Fonts via @import)
- Garantir que a capa do livro carregue com crossOrigin="anonymous" para evitar erro de CORS no canvas

## FUNCIONALIDADES TÉCNICAS ADICIONAIS

### Scan de código de barras (ISBN)
- Usar biblioteca html5-qrcode
- Ao abrir, ativar câmera traseira
- Detectar código de barras EAN-13 (formato ISBN)
- Ao detectar, buscar na Google Books API automaticamente
- Feedback visual: quadro verde ao detectar, vibração do dispositivo
- Fallback: botão para digitar ISBN manualmente

### PWA
- manifest.json com:
  - name: "Leio"
  - short_name: "Leio"
  - theme_color: "#1B4332"
  - background_color: "#FAFAF5"
  - display: "standalone"
  - orientation: "portrait"
  - icons: múltiplos tamanhos (192, 512)
- Service worker para cache de assets estáticos
- Prompt de "Adicionar à tela inicial"

### Timer resiliente
- Usar Web Worker para manter o timer preciso mesmo com a aba em background
- Salvar timestamp de início no localStorage
- Ao retornar à aba, calcular duração real por diferença de timestamps (não confiar no setInterval)
- Exibir tempo em MM:SS para sessões < 1h, HH:MM:SS para sessões maiores

### Offline-first
- Sessões de leitura devem funcionar sem internet
- Dados salvos localmente, sincronizar quando voltar online
- Busca de livros requer internet (mostrar livros já salvos offline)

## OBSERVAÇÕES FINAIS

- Toda a interface deve estar em português do Brasil
- Responsivo, mas priorizar experiência mobile (90% dos usuários vão usar no celular)
- Animações suaves (Framer Motion ou CSS transitions) — o app precisa sentir-se premium
- Cada interação importante deve ter micro-feedback (vibração, som sutil, animação)
- Acessibilidade: cores com contraste suficiente, labels em todos os inputs, navegação por teclado
- Performance: lazy loading de imagens, virtualização de listas longas
- Todas as datas e horários devem considerar timezone de Brasília (UTC-3)
- Mensagens de erro amigáveis e em português: "Ops, não encontramos esse livro" em vez de "Error 404"
- O card compartilhável DEVE ter a marca "Leio" visível (mas sutil) — cada compartilhamento é marketing
