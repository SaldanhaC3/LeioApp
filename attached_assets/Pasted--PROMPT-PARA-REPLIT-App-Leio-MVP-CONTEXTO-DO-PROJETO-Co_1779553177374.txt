# PROMPT PARA REPLIT — App "Leio" (MVP)

---

## CONTEXTO DO PROJETO

Construa um aplicativo web mobile-first (PWA) chamado **"Leio"** — um app de leitura gamificado inspirado no Strava. O conceito central: transformar sessões de leitura em conquistas visuais compartilháveis nas redes sociais. O público é brasileiro, tudo deve estar em português do Brasil.

A proposta de valor é: "Se não registrou, não leu." O app deve tornar a leitura um ato social e visualmente orgulhoso, não uma atividade solitária e invisível.

O diferencial competitivo em relação ao Bookly (concorrente direto) é: a imagem compartilhável é o produto, não uma feature secundária. O card de conclusão de sessão precisa ser tão bonito e reconhecível que funcione como marketing orgânico toda vez que alguém postar no Instagram/Stories.

---

## STACK TÉCNICA

- **Frontend:** React + TypeScript + Tailwind CSS
- **Estrutura:** PWA (Progressive Web App) com manifest.json e service worker
- **Backend:** Node.js + Express (ou Fastify)
- **Banco de dados:** SQLite (via better-sqlite3 ou Prisma com SQLite) para o MVP
- **API de livros:** Google Books API (gratuita, busca por ISBN e título, retorna capa, autor, gênero, número de páginas)
- **Geração de imagem:** html2canvas ou dom-to-image para gerar o card compartilhável como PNG
- **Scan de código de barras:** biblioteca QuaggaJS ou html5-qrcode para leitura de ISBN via câmera
- **Armazenamento de imagens:** Local/base64 no MVP
- **Autenticação:** Simples com email + senha (bcrypt + JWT) — permitir uso sem login até o momento de compartilhar

## IDENTIDADE VISUAL — "Collins editorial meets Nike performance"

A estética do Leio combina duas referências: a sofisticação editorial da Collins (tipografia forte, whitespace generoso, hierarquia limpa, layout de revista premium) com a energia esportiva da Nike (dados de performance em destaque, linguagem de treino, contraste preto/branco bold, sensação de conquista atlética). A leitura é tratada como esporte mental.

### Direção de design
- Layout tipo editorial de revista de alta qualidade (referência: wearecollins.com): muito espaço em branco, tipografia como elemento principal, composição assimétrica com intenção
- Dados de performance tratados como a Nike Run Club trata: números grandes, bold, monospace, com destaque absoluto. O pace (pág/min) deve ter a mesma presença visual que o km/h tem no Nike Run Club
- Preto, branco e um accent neon/volt como a Nike usa. Nada de paletas tímidas
- Cards compartilháveis devem parecer posters esportivos de alta qualidade, não infográficos de planilha
- Zero ornamento desnecessário. Cada elemento existe por um motivo
- Sensação geral: "esse app me faz sentir que ler é uma coisa séria e bonita ao mesmo tempo"

### Paleta de cores
- **Background principal:** #FAFAF8 (off-white quente, como papel de livro de alta qualidade)
- **Background escuro (dark surfaces/cards):** #0A0A0A (preto profundo, para contraste dramático)
- **Texto primário:** #0A0A0A
- **Texto secundário:** #6B6B6B
- **Texto sobre fundo escuro:** #FAFAF8
- **Accent principal:** #CDFF00 (volt/neon green, referência Nike, usado com parcimônia: CTAs, pace em destaque, badges ativos, progress bars)
- **Accent secundário:** #FF5733 (coral quente, para alertas de streak perdido, notificações urgentes, fôlego em risco)
- **Surface cards (light):** #FFFFFF com border 1px #E5E5E5
- **Surface cards (dark):** #1A1A1A com border 1px #2A2A2A
- **Progress bars/rings:** gradient de #CDFF00 para #00E676

### Modo escuro
O app deve ter toggle de tema claro/escuro. O dark mode inverte: background #0A0A0A, cards #1A1A1A, texto #FAFAF8. O accent volt (#CDFF00) funciona perfeitamente em ambos os modos. O dark mode é o modo "padrão" (como a Nike Run Club), pois a maioria dos leitores lê à noite. O onboarding pergunta a preferência.

### Tipografia
- **Display/headlines (números de performance, pace, stats grandes):** Space Grotesk (bold, geométrica, atlética, moderna). Alternativa: Clash Display ou Satoshi.
  - Pace no timer: 64px, bold
  - Stats grid: 32px, bold
  - Headlines editoriais: 40-48px, medium
- **Body/UI (labels, descrições, navegação):** Inter Tight ou DM Sans (clean, legível, amigável, UI-grade)
  - Body: 15px, regular
  - Labels/captions: 12px, medium, uppercase tracking +0.05em (como a Nike faz)
- **Dados técnicos nos cards (pace, páginas, tempo):** JetBrains Mono (monospace, reforça a sensação de "dado medido com precisão")
  - Dados nos cards: 18-24px, medium
- **Quote nos cards:** Newsreader ou Lora (serif editorial, contrasta com o resto da tipografia atlética)
  - Quotes: 16-18px, italic

### Labels e linguagem visual (estilo Nike)
- Labels de categorias e seções em UPPERCASE com tracking amplo: "SESSÃO ATIVA", "SEU PACE", "FÔLEGO", "BIBLIOTECA", "METAS"
- Números de performance sempre em monospace bold com unidade em texto menor ao lado: "1.3 pág/min", "42 págs", "35 min"
- Barras de progresso finas (2-3px) e neon (#CDFF00), nunca grossas ou arredondadas demais
- Ícones minimalistas, outline, stroke 1.5px (estilo Phosphor Icons ou Lucide)
- Zero uso de emojis na UI principal (emojis permitidos apenas nos badges e nas mensagens de tom de voz)

### Tom de voz
- Combina motivação esportiva com humor brasileiro casual
- Referências ao esforço mental como esporte: "Treino encerrado", "Pace da sessão", "Fôlego" (streak), "Personal best"
- Encorajador como um coach de corrida: "42 páginas em 35 min. Tá em forma!", "Não perde o fôlego agora!"
- Humor brasileiro genuíno (sem exagero): "Cê vai ler hoje ou não vai?", "Ô, sumido. Vai deixar o fôlego acabar?", "Só 10 minutinhos já conta, viu?"
- Mensagens de conclusão variam: "Treino mental concluído.", "Mente em forma.", "Sessão encerrada. Respeito."
- Evitar: linguagem infantil, excesso de emojis, gírias forçadas, euforia artificial

### Cards compartilháveis — direção visual
Os cards são a peça de marketing mais importante. Devem parecer posters esportivos editoriais, não infográficos corporativos.

Princípios:
- Tipografia grande e bold como elemento principal (como um poster da Nike ou da Collins)
- Dados de performance com presença visual dominante (pace em 72px, por exemplo)
- Capa do livro como elemento secundário/de apoio, nunca o elemento principal do layout
- Contraste forte: fundo escuro com texto claro, ou fundo de imagem com overlay escuro
- Logo "Leio" pequeno e discreto, como uma marca de roupa esportiva no canto
- O card deve ser reconhecível à distância: quando alguém vir nos Stories, deve saber que é do Leio antes de ler qualquer texto

### Composição geral das telas
- Whitespace agressivo (mais espaço do que conteúdo, como a Collins)
- Elementos alinhados em grid de 4 colunas no mobile
- Cards sem border-radius exagerado: 8px máximo (nada de bolhas ou pílulas)
- Sombras: zero. Separação por espaço e linhas finas quando necessário
- Animações: rápidas (200-300ms), ease-out, sem bounce. Transições de página tipo slide horizontal
- Bottom tab bar: minimalista, ícones outline, label em uppercase 10px, accent no item ativo

### 3. TELA DE NOVA SESSÃO — SELECIONAR LIVRO
Se o usuário já tem um livro em leitura, pré-selecionar automaticamente.

**Opções para adicionar livro:**
- Campo de busca por título/autor (consulta Google Books API)
- Botão "Escanear código de barras" (abre câmera, lê ISBN, busca na API)
- Lista dos livros já na biblioteca do usuário

**Card de resultado da busca:**
- Capa do livro (thumbnail da API)
- Título, autor, número de páginas, gênero/categoria
- Botão "Selecionar este livro"

**Ao selecionar:**
- Perguntar "Em que página você está?" (campo numérico)
- Livro adicionado à biblioteca automaticamente
- Ir para a tela de sessão ativa

### 4. TELA DE SESSÃO ATIVA (o coração do app)
Design inspirado no timer do Strava durante atividade.

**Layout:**
- Capa do livro no topo (pequena, com título e autor)
- Timer grande no centro (MM:SS), fonte monospace, animação de pulsação sutil a cada segundo
- Pace em tempo real abaixo do timer: "0.0 pág/min" (atualiza conforme o usuário registra páginas)
- Botão "Atualizar página atual" — abre um input numérico rápido para o usuário informar em que página está (cada atualização recalcula o pace em tempo real)
- Página inicial da sessão (fixa, definida no início)
- Página atual (editável durante a sessão)
- Páginas lidas na sessão (calculado automaticamente: página atual - página inicial)

**Controles:**
- Botão "Pausar" (pausa o timer, permite retomar)
- Botão "Encerrar sessão" (finaliza e vai para a tela de geração do card)
- O timer deve funcionar mesmo com a tela bloqueada/em background (usar Web Workers ou service worker para manter contagem)

**Informação secundária (colapsável):**
- Progresso geral do livro (barra de progresso)
- Estimativa de tempo para concluir o livro baseada no pace atual

### 5. TELA DE CONCLUSÃO DA SESSÃO — GERAÇÃO DO CARD COMPARTILHÁVEL
Esta é a tela mais importante do app. É o momento "troféu".

**Resumo da sessão (topo):**
- Animação de entrada dos dados (números contando de 0 até o valor final, tipo placar)
- Páginas lidas: XX
- Tempo de leitura: XX min
- Pace: X.X pág/min
- Progresso do livro: XX%

**Se o usuário desbloqueou um badge nesta sessão:**
- Animação especial de badge desbloqueado (confetti + badge girando)
- Nome do badge e descrição

**Geração do card compartilhável:**

O card é uma imagem de 1080x1920px (formato Stories) ou 1080x1080px (formato feed).

**Elementos do card:**
- Imagem de fundo: o usuário pode escolher entre:
  - Foto tirada na hora (câmera)
  - Foto da galeria
  - Template pré-definido (5 opções com cores/texturas diferentes: madeira, biblioteca, café, noturno, minimalista)
- Overlay semi-transparente escuro para legibilidade
- Logo "Leio" pequeno no canto superior (marca d'água sutil)
- Capa do livro (com sombra, leve rotação 3D)
- Título e autor do livro
- Dados da sessão em layout limpo:
  - ⏱️ Tempo: 45 min
  - 📖 Páginas: 32
  - ⚡ Pace: 0.7 pág/min
  - 📊 Progresso: 67%
- Badge conquistado (se houver)
- Frase/quote opcional (campo de texto que o usuário pode preencher com um trecho do livro que o marcou)
- Data da sessão

**Personalização do card:**
- Escolha de layout: "Clássico" (vertical, capa grande), "Minimalista" (dados em destaque, capa pequena), "Editorial" (parece página de revista)
- Escolha de cor do overlay (escuro, claro, sépia)

**Botões de ação:**

## LÓGICA DE NEGÓCIO IMPORTANTE

### Cálculo do Pace
- Pace = páginas lidas ÷ (duração em minutos)
- Exibir com 1 casa decimal: "1.3 pág/min"
- Pace em tempo real durante a sessão: atualizar a cada vez que o usuário informar a página atual
- Pace médio do livro: média ponderada por duração de todas as sessões do livro
- Pace médio geral: média ponderada de todas as sessões do usuário

### Cálculo do Streak
- Após cada sessão, verificar se o usuário leu no dia anterior
- Se sim, incrementar streak_current
- Se não (pulou um dia), resetar streak_current para 1
- Atualizar streak_max se streak_current > streak_max
- Considerar o dia pela data local do usuário (timezone Brasil: UTC-3)

### Verificação de Badges
Após cada sessão salva, rodar a verificação completa de badges:
1. Verificar todos os critérios de badges que o usuário ainda não desbloqueou
2. Para cada badge desbloqueado, salvar em user_badges com o session_id que triggered
3. Retornar lista de badges recém-desbloqueados para a tela de conclusão

### Busca de livros (Google Books API)
- Endpoint: `GET https://www.googleapis.com/books/v1/volumes?q={query}`
- Para ISBN: `GET https://www.googleapis.com/books/v1/volumes?q=isbn:{isbn}`
- Extrair: title, authors[0], pageCount, categories[0], imageLinks.thumbnail, industryIdentifiers
- Se thumbnail vier como http, converter para https
- Tratar casos onde pageCount é undefined (pedir ao usuário para informar manualmente)

### Geração do card compartilhável
- Renderizar um componente React invisível (off-screen) com o layout do card
- Usar html2canvas para converter em PNG
- Tamanho: 1080x1920 (Stories) e 1080x1080 (Feed)
- O componente do card deve usar posicionamento absoluto e fontes inline (Google Fonts via @import)
- Garantir que a capa do livro carregue com crossOrigin="anonymous" para evitar erro de CORS no canvas

## FUNCIONALIDADES TÉCNICAS ADICIONAIS

### Scan de código de barras (ISBN)
- Usar biblioteca html5-qrcode
- Ao abrir, ativar câmera traseira
- Detectar código de barras EAN-13 (formato ISBN)
- Ao detectar, buscar na Google Books API automaticamente
- Feedback visual: quadro verde ao detectar, vibração do dispositivo
- Fallback: botão para digitar ISBN manualmente

### PWA
- manifest.json com:
  - name: "Leio"
  - short_name: "Leio"
  - theme_color: "#0A0A0A"
  - background_color: "#FAFAF8"
  - display: "standalone"
  - orientation: "portrait"
  - icons: múltiplos tamanhos (192, 512)
- Service worker para cache de assets estáticos
- Prompt de "Adicionar à tela inicial"

### Timer resiliente
- Usar Web Worker para manter o timer preciso mesmo com a aba em background
- Salvar timestamp de início no localStorage
- Ao retornar à aba, calcular duração real por diferença de timestamps (não confiar no setInterval)
- Exibir tempo em MM:SS para sessões < 1h, HH:MM:SS para sessões maiores

### Offline-first (robusto)
- Sessões de leitura devem funcionar sem internet
- IndexedDB (via Dexie.js ou idb-keyval) como store local para livros, sessões e spots
- Service worker intercepta chamadas API POST/PATCH: em caso de falha, enfileira no IndexedDB e retenta ao reconectar
- Barcode scan salva ISBN localmente de imediato; sincroniza detalhes do livro quando conectar
- Registro manual de livro funciona offline: título + autor + páginas salvos localmente
- Indicador de status de sync na UI: "X itens aguardando sync" com ícone sutil
- Busca de livros via Google Books requer internet (mostrar livros já salvos offline)

### "Fôlego" — Sistema de streak renomeado
O streak de dias consecutivos lendo se chama "Fôlego" (fôlego = breath/stamina).
- Mensagens: "Você tá com X dias de fôlego", "Não perde o fôlego agora!"
- Ao perder o streak: "Acabou o fôlego... mas bora de novo!"
- O ícone de streak é um ícone de vento/fôlego, não fogo (diferenciação do Strava/Duolingo)
- A palavra "Fôlego" substitui "streak" em toda a interface

### Efeitos sonoros
Sons curtos que reforçam micro-interações (todos opcionais, mutados por padrão, opt-in nas configurações):
- Início de sessão: som suave de virar página
- Milestone do timer (a cada 15 min): chime discreto
- Badge desbloqueado: fanfarra curta e divertida
- Fôlego aumentado: som de "whoosh" (sopro)
- Fim de sessão: "thud" satisfatório de livro fechando
- Compartilhar card: clique de câmera
- Implementar via Web Audio API com fallback HTML Audio
- Toggle global de mute no settings, estado salvo em localStorage

### Push Notifications (Web Push API)
Notificações para manter engajamento diário:
- **Nudge de Fôlego:** Se o usuário não leu hoje até horário configurável (padrão 21h), enviar mensagem rotativa: "Ô, sumido. Vai deixar o fôlego acabar?", "Tá passando da hora, hein", "Só 10 minutinhos já conta, viu?"
- **Deadline de meta:** 24h antes do fim do período de meta semanal/mensal se não atingida: "Faltam X páginas pra bater a meta. Dá tempo ainda!"
- **Badge quase lá:** Quando perto de desbloquear: "Você tá a 3 sessões de virar Maratonista. Bora?"
- VAPID keys no backend; service worker exibe notificações
- Usuário configura horário do nudge nas configurações
- Permissão pedida de forma contextual (após 3ª sessão, não no primeiro acesso)

### Ícone adaptativo por horário (PWA)
O ícone do app na tela inicial muda conforme o horário do dia (7 variantes SVG/PNG):
- 06h-09h: nascer do sol + livro aberto
- 09h-12h: xícara de café + livro
- 12h-14h: banco de parque + livro (leitura na hora do almoço)
- 14h-18h: sol da tarde + livro
- 18h-21h: janela de ônibus/metrô + livro (hora do commute)
- 21h-00h: lua + livro (coruja noturna)
- 00h-06h: estrelas + livro (insone por escolha)
- Implementado via JS atualizando <link rel="icon"> e manifest icons no page load e periodicamente

### "Modo Vagão" — modo leitura para transporte público
Modo especial para quem lê no ônibus/metrô (condições erráticas):
- Botão "Modo Vagão 🚇" rápido na home e na tela de início de sessão
- UI com botões oversized: "+" e "-" para atualizar página com um toque do polegar em ônibus trepidando, sem abrir teclado
- Auto-pause por shake: DeviceMotion API detecta movimento brusco (freada, solavanco) e pausa o timer, retomando quando estabiliza
- Botão "Cheguei" de um toque para encerrar rápido quando chegar no ponto, sem fluxo de confirmação normal
- Lock de orientação portrait para conforto em veículos lotados
- Mensagens de conclusão específicas: "Chegou e ainda leu X páginas. Isso é poder!", "Vai no ônibus e ainda dá conta. Respeito."

### "Cantinhos de Leitura" — pontos de leitura no mapa
Usuários podem compartilhar e descobrir bons lugares para ler:
- Após concluir sessão, prompt opcional: "Esse foi um bom lugar pra ler? Marca no mapa!"
- Aba "Cantinhos" no app mostrando lista/mapa com spots contribuídos por usuários
- Cada spot: foto opcional, pin de localização, nome definido pelo usuário (ex: "Parque do Ibirapuera, banco do lago"), tags (silencioso, ar livre, com tomada, bom wifi, café perto), nota breve
- Botão "Bom cantinho!" para upvote
- Spots são públicos e navegáveis sem login
- Submissions offline ficam em fila e sincronizam quando online
- MVP: lista de cards com coordenadas + embed simples de mapa (mapa completo é futuro)

### 13. IMPORTAÇÃO DO GOODREADS

Muitos leitores brasileiros já têm estante no Goodreads. Oferecer importação reduz a fricção de "começar do zero".

**Fluxo:**
- O Goodreads permite exportação via CSV (Settings > Import/Export > Export Library)
- O usuário faz o download do CSV no Goodreads e faz upload no Leio
- O parser do CSV extrai: título, autor, ISBN, número de páginas, rating, data de leitura, estante (read/currently-reading/to-read)
- Para cada livro, buscar capa na Google Books API via ISBN
- Mapear estantes do Goodreads para status do Leio:
  - "read" → "Lido"
  - "currently-reading" → "Lendo"
  - "to-read" → "Quero ler"
- Exibir resumo antes de confirmar: "Encontramos X livros. Importar?"
- Após importação, verificar badges que podem ser desbloqueados retroativamente

**Campos do CSV do Goodreads usados:**
- Title, Author, ISBN13, Number of Pages, My Rating, Date Read, Exclusive Shelf, Date Added

### 14. IMPORTAÇÃO DE HIGHLIGHTS DO KINDLE

### 17. INTEGRAÇÃO COM ACERVO GRATUITO DO MEC LIVROS E DOMÍNIO PÚBLICO

O Governo Federal lançou em abril de 2026 o MEC Livros (meclivros.mec.gov.br), uma biblioteca digital gratuita com 25 mil títulos (clássicos brasileiros, internacionais e contemporâneos licenciados). Acesso via conta gov.br. O Portal Domínio Público (dominiopublico.gov.br) existe desde 2004 com quase 200 mil obras de domínio público com download direto em PDF.

Nenhum deles oferece API pública. A estratégia é curadoria interna: manter uma tabela própria de livros gratuitos com links diretos para as plataformas.

**Tabela: free_books (curadoria interna)**
```
id (UUID, PK)
title (string)
author (string)
genre (string)
total_pages (integer)
cover_url (string, nullable — via Google Books API)
source (enum: 'mec_livros', 'dominio_publico', 'gutenberg_pt')
source_url (string — link direto para a obra na plataforma de origem)
isbn (string, nullable)
created_at (datetime)
featured (boolean, default false)
```

**Como usar no app:**

1. **Tag "Grátis" nos livros da biblioteca:**
   - Ao adicionar um livro via busca, verificar se ele existe na tabela free_books (match por título+autor ou ISBN)
   - Se existir, exibir badge "📗 Disponível grátis no MEC Livros" com link direto
   - Isso funciona como discovery: o usuário descobre que pode ler de graça

2. **Desafios temáticos com livros gratuitos:**
   - Criar desafios exclusivos com curadoria de livros do MEC Livros/Domínio Público
   - Exemplos:
     - "Clássicos Brasileiros Grátis" — Ler 3 obras de Machado de Assis, Lima Barreto ou Aluísio de Azevedo (todas disponíveis no MEC Livros)
     - "Domínio Público Challenge" — Ler 5 livros disponíveis gratuitamente

   - Cada desafio lista os livros elegíveis com link direto para leitura gratuita
   - Badge exclusivo ao completar

3. **Seção "Estante Gratuita" na tela de Biblioteca:**
   - Aba adicional "Grátis" mostrando livros curados disponíveis gratuitamente
   - Filtro por gênero
   - Cada livro com botão "Ler no MEC Livros" (abre link externo) e botão "Adicionar à minha biblioteca" (para rastrear no Leio)

4. **Destaque na Home:**

## OBSERVAÇÕES FINAIS

- Toda a interface deve estar em português do Brasil
- Responsivo, mas priorizar experiência mobile (90% dos usuários vão usar no celular)
- Animações suaves (Framer Motion ou CSS transitions) — o app precisa sentir-se premium
- Cada interação importante deve ter micro-feedback (vibração, som sutil, animação)
- Acessibilidade: cores com contraste suficiente, labels em todos os inputs, navegação por teclado
- Performance: lazy loading de imagens, virtualização de listas longas
- Todas as datas e horários devem considerar timezone de Brasília (UTC-3)
- Mensagens de erro amigáveis e em português: "Ops, não encontramos esse livro" em vez de "Error 404"
- O card compartilhável DEVE ter a marca "Leio" visível (mas sutil) — cada compartilhamento é marketing
- Cada card compartilhado deve conter URL curta ou QR code do app — todo share é aquisição
- O campo de quote no card é opcional, mas deve ser incentivado (ex: "Alguma frase te marcou nessa sessão?")
- A integração com Kindle é via upload de arquivo (My Clippings.txt), não via API — Amazon não oferece API oficial
- A importação do Goodreads é via CSV, não via API — Goodreads descontinuou a API pública
- A integração com MEC Livros é via curadoria interna (tabela free_books) e links diretos, não via API — o MEC não oferece API pública
- Desafios com livros gratuitos do MEC devem ser priorizados como diferencial competitivo e narrativa de impacto social
- O selo "Li de graça no MEC Livros" no card compartilhável serve como divulgação orgânica do acervo público e posiciona o Leio como aliado da educação
